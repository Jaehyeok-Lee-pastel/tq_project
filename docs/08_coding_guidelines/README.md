# 코딩 가이드 (Coding Guidelines)

상태: approved (template) · 기준일: 2026-06-02

## 목적

이 폴더는 프로젝트 개발 시 항상 참조할 코딩 규약을 관리한다. Python/FastAPI, React/TypeScript, 파일 인코딩, 네이밍, 폴더 구조, 테스트, 리뷰 기준을 한곳에 둔다.

## 기준 원칙

- 자동 포맷터·린터가 판단할 수 있는 규칙은 도구에 맡긴다.
- 사람이 판단해야 하는 규칙은 작게 유지하고, 읽기 쉬움과 변경 안전성을 우선한다.
- 보안·명확성·수정 가능성을 기본값으로 둔다(특히 사용자 데이터·AI 결과·시크릿).
- 모든 소스/문서는 UTF-8로 저장한다.
- 기능은 도메인 단위로 모으고, 공통 코드는 필요한 시점에만 추출한다.

## 문서 목록

1. `01_general_rules.md` — 공통 코딩 원칙, 네이밍, UTF-8, 주석, 에러 처리, 보안
2. `02_python_fastapi.md` — Python/FastAPI 구조, 타입힌트, 서비스/라우터 분리
3. `03_react_typescript.md` — React/TypeScript 구조, 컴포넌트, hooks, API 호출, 스타일
4. `04_project_structure.md` — 모노레포 폴더 구조와 파일 배치 규칙
5. `05_quality_checklist.md` — 구현 전/후 체크리스트, 리뷰 기준

> `06_current_code_structure.md`는 **프로젝트별로** 현재 REST API·폴더 스냅샷을 기록할 때 추가한다(템플릿에는 없음).

## 우선순위 (충돌 시)

1. 이미 코드에 적용된 로컬 패턴
2. 이 코딩 가이드
3. 공식 문서와 도구 기본값
4. 일반 취향

로컬 코드와 가이드가 충돌하면, 작은 변경에서는 로컬 패턴을 따르고 큰 구조 변경 때 가이드를 업데이트한다.

> Claude는 `.claude/rules/stack-*.md` 요약 카드로 이 규칙을 항상 로드한다.
