# TQ Coach 설계 검증 및 작업 순서

작성일: 2026-07-03

## 앱의 핵심 철학

TQ Coach는 TQQQ를 단순 추천하는 앱이 아니라, QQQ 200일 이동평균선을 기준 신호로 삼아 공격 자산을 조건부로 운용하는 개인 투자 전략 코치앱이다. QQQ는 판단 기준, TQQQ/QLD는 실행 자산, 현금/SGOV/BIL은 분할매수 대기자산으로 구분한다.

전략 추천은 예측이 아니라 규칙 기반이어야 한다. AI는 설명, 요약, 조정 조언에 사용하고, 가격 데이터와 매수/매도 판단은 검증 가능한 규칙과 시장 데이터에서 나온다.

## 현재 수준 평가

- 전략 철학 반영: 8/10
- TQQQ 200일선 운용 규칙 반영: 8/10
- 일일 실행 가이드: 7/10
- 월급 추가금 및 현금 흐름 코칭: 6.5/10
- 백테스트/모의운용 연결: 6/10
- 신뢰도/검증 체계: 6/10
- UX 정보 구조: 6/10

전체적으로 MVP+ 또는 초기 베타 수준이다. 기능의 방향은 맞지만, 전략관리 화면이 많은 역할을 동시에 맡고 있어 일일 운용 흐름 기준으로 재정리가 필요하다.

## 권장 화면 책임

### 전략 수립

현재 보유 종목, 현금, 리스크 성향, 목표 종목 수, 최소 현금, 최대 TQQQ 비중을 입력하고 검증된 ETF 후보군 안에서 전략을 추천받는다.

### 전략 관리

채택한 전략을 매일 운용하는 중심 화면이다. 오늘 판단, 실행 기록, 추가금/조정, 목표 규칙, 버전 이력을 순서대로 제공한다.

### 테스트/모의 보유

같은 원금으로 여러 전략의 과거 성과와 실시간 모의 보유 성과를 비교한다. 최종 목표는 전략 선택 전후의 검증이다.

### 전략 비교

여러 전략 후보를 동일 기준으로 비교한다.

### ETF 후보군

검증된 ETF 후보군, 포함/제외 근거, TQQQ 200일선 전략과의 궁합을 관리한다.

### 규칙/신뢰도

전략의 근거, 데이터 신뢰도, 규칙 검증 항목, 주의사항을 관리한다.

## 전략관리 탭 권장 순서

1. 오늘의 판단
2. 실행 기록
3. 조정/추가금
4. 목표/규칙
5. 버전 이력

사용자가 매일 앱에 들어왔을 때 가장 먼저 필요한 것은 "오늘 매수, 매도, 대기 중 무엇을 해야 하는가"이다. 그 다음 실제 실행을 기록하고, 필요할 때 추가금이나 현금 비중 조정을 검토한다. 목표 규칙과 버전 이력은 참고 및 감사 목적이다.

## 진행 순서

1. 전략관리 탭 순서를 일일 운용 흐름에 맞게 재배열한다.
2. "실행 현황" 탭을 "오늘의 판단"으로 명확히 바꾼다.
3. "전략 조정" 탭을 "조정/추가금"으로 바꾸고 월급 추가금 코치와 현금 비중 조정을 같은 맥락으로 묶는다.
4. 실행 기록 탭을 오늘 판단 다음에 배치해 "판단 -> 기록" 흐름을 강화한다.
5. 목표/규칙 탭은 정적 기준과 원 규칙 중심으로 유지하고, 실시간 판단은 오늘의 판단 쪽으로 점진적으로 이동한다.
6. 전략관리 화면의 임시 레거시 블록을 제거하고 타입을 정상화한다.
7. 빌드 검증을 통해 타입 오류와 번들 오류를 확인한다.

## 다음 고도화 권장 순서

1. 데이터 신뢰도 패널 추가
   - QQQ 지표 기준일
   - 최신 데이터 여부
   - 장중/종가 구분
   - 가격 API 제공자

2. 현금 흐름 원장 추가
   - 월급 입금
   - 환전
   - SGOV/BIL 대기 운용
   - 매수 재원 전환

3. 핵심 규칙 테스트 추가
   - QQQ 200일선 아래 매수 금지
   - 20일/50일/신고가 조건 판정
   - 1차/2차/3차 누적 집행률 계산
   - SGOV/BIL 현금성 자산 처리
   - 추가금 코치가 자동매수가 아닌 선택 가능한 실행안으로 동작

4. 저장 전략과 백테스트/모의 보유 연결 강화
   - 선택한 저장 전략을 바로 백테스트 실행
   - 저장 전략으로 모의 보유 시작
   - 원금 동일 비교

## 완료 기준

- 전략관리 탭 순서가 일일 운용 흐름과 일치한다.
- 사용자가 첫 화면에서 오늘 판단과 다음 실행 기준을 확인할 수 있다.
- 실행 기록이 오늘 판단 다음 단계로 자연스럽게 연결된다.
- 추가금 코치는 바로 적용하지 않고 여러 실행안 중 선택 후 적용한다.
- 전략 원 규칙과 버전 이력이 분리되어 신뢰도와 추적성이 유지된다.
- 프론트엔드 빌드가 통과한다.
