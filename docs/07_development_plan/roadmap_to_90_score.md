# TQ Coach 90점 고도화 로드맵

작성일: 2026-07-03

## 목표

현재 앱은 TQQQ 200일선 매매법 기반 개인 전략 코치앱으로서 약 72점 수준이다. 90점 수준의 목표는 단순 추천 앱이 아니라, 같은 원금으로 QQQ/QLD/TQQQ 전략을 연구하고, 백테스트하고, 실시간 모의운용하고, 실제 실행 기록까지 관리하는 개인 전략 연구소 겸 운용 코치다.

## 90점의 정의

- 현재 전략을 만들 수 있다.
- 저장 전략을 그대로 백테스트할 수 있다.
- 같은 원금으로 여러 전략을 비교할 수 있다.
- 실제 매수 전 모의운용할 수 있다.
- 매수 후 기록과 리뷰를 받을 수 있다.
- 데이터가 믿을 만한지 확인할 수 있다.
- 추천 전략의 근거와 위험을 확인할 수 있다.
- 감정적 추격매수를 앱이 막아준다.

## 앱 철학

TQ Coach는 TQQQ를 많이 사게 만드는 앱이 아니다. QQQ 200일선을 기준 신호로 삼고, QQQ가 장기 추세 위에 있을 때만 TQQQ/QLD 같은 레버리지 자산을 제한적으로 사용한다. 현금, SGOV, BIL은 단순 방치 자산이 아니라 분할매수와 방어 전환을 위한 대기자산이다.

## 고도화 순서

### 1. 데이터 신뢰도 90점화

- QQQ/QLD/TQQQ/SGOV 데이터 상태 API 제공
- 최신 기준일, 데이터 나이, 20/50/200일선 계산 가능 여부 표시
- 장중 가격과 종가 기반 신호 구분
- API 실패 시 마지막 정상 데이터와 경고 표시
- 프론트 판단 패널에서 데이터 신뢰도 점수 표시

완료 기준: 사용자가 오늘 판단이 어떤 데이터 기준인지 명확히 알 수 있다.

### 2. 저장 전략 기반 백테스트

- 전략관리에서 저장 전략을 그대로 백테스트 실행
- TQQQ/QLD 목표 비중, 현금 수익률, 200일선 기준을 자동 반영
- 기존 단순 `tqqq_200ma`, `qld_200ma`와 저장 전략 백테스트 결과 비교
- 1차/2차/3차/재장전 규칙 포함 백테스트로 확장

완료 기준: 저장한 전략을 과거 데이터로 직접 검증할 수 있다.

### 3. QQQ/QLD/TQQQ 전략 연구실

- QQQ 기준 200일선 전략
- TQQQ 기준 200일선 전략
- QLD 기준 200일선 전략
- 이격도 5/8/15% 민감도 테스트
- 분할 비율 30/35/35, 20/30/50 등 비교
- SGOV 대기 vs 현금 대기 비교

완료 기준: 같은 원금으로 여러 규칙을 실험하고 비교할 수 있다.

### 4. 모의운용 고도화

- 저장 전략으로 모의운용 시작
- 가상 체결가, 수량, 환율 기록
- 현재가 갱신 시 평가금액과 목표비중 이탈 표시
- SGOV 매도 후 TQQQ 매수 재원 전환 시뮬레이션
- 월급 추가금 모의 반영

완료 기준: 실제 매수 전 전략별 실시간 모의계좌를 운용할 수 있다.

### 5. 전략 신뢰도 점수 체계

- 데이터 신뢰도
- 규칙 명확성
- 과최적화 위험
- 최대낙폭 방어
- 실행 난이도
- 사용자 성향 적합도
- 현재 시장 적합도

완료 기준: 추천 전략의 신뢰도와 약점을 수치와 문장으로 함께 확인할 수 있다.

### 6. 실행 후 리뷰 시스템

- 매수/매도 후 규칙 준수 여부 자동 평가
- 체결가와 기준가 차이 분석
- 감정 메모와 성과 연결
- 실수 패턴 감지
- 다음 액션 추천

완료 기준: 기록장이 사후 학습 코치 역할을 한다.

### 7. 리스크 예산 관리

- TQQQ 최대 비중
- 레버리지 노출 총량
- 현금/SGOV 최소 비중
- 1회 매수 최대 금액
- 월급 추가금 중 공격자산 투입 한도
- 3차 이후 재장전 한도
- 200일선 이탈 시 강제 방어 규칙

완료 기준: 공격적이지만 무너지지 않는 운용 한도가 자동 점검된다.

### 8. UX 구조 재배치

- 전략 수립
- 전략 연구실
- 백테스트
- 모의운용
- 전략 관리
- 실행 기록
- 규칙/신뢰도
- ETF 후보군

완료 기준: 각 화면의 책임이 명확하고 한 탭에 너무 많은 기능이 몰리지 않는다.

### 9. 근거 아카이브

- TQQQ 200일선 매매법 근거
- QQQ 기준선을 쓰는 이유
- QLD와 TQQQ를 같이 들지 않는 이유
- SGOV를 대기자산으로 쓰는 이유
- 3차 이후 재장전 제한 이유
- 레버리지 ETF 위험 고지
- 백테스트 해석 주의사항

완료 기준: 앱의 조언을 사용자가 납득할 수 있다.

### 10. 테스트 커버리지 확장

- 200일선 아래 매수 금지
- +15% 이상 신규매수 금지
- 2차/3차 순서 제한
- 재장전 조건
- SGOV 재원 전환
- 월급 추가금 적용
- 저장 전략 백테스트 연결
- 모의운용 평가금액 갱신
- 데이터 실패 시 경고

완료 기준: 전략 규칙이 코드 변경으로 느슨해지지 않는다.

## 우선순위

1. 데이터 신뢰도 API와 프론트 패널 연결
2. 저장 전략 기반 백테스트 실행
3. 전략 연구실 기본 화면
4. 모의운용 저장 전략 연결
5. 신뢰도 점수 체계
6. 실행 후 리뷰
